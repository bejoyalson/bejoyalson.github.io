Implementing React Performance Budgets

Published on October 25, 2025 by Bejoy Alson

Why Bundle Size Matters

A 1MB JavaScript bundle on a 4G connection takes ~4 seconds to download. That's 4 seconds before your app even starts rendering. Performance budgets prevent code bloat by enforcing size limits and alerting developers when they're exceeded.

Setting Up with webpack-bundle-analyzer

First, visualize your bundle: 

npm install --save-dev webpack-bundle-analyzer

// In webpack.config.js or next.config.js
const BundleAnalyzerPlugin = require('webpack-bundle-analyzer').BundleAnalyzerPlugin;

module.exports = {
  plugins: [
    new BundleAnalyzerPlugin({
      reportFilename: 'bundle-report.html'
    })
  ]
};

Run your build and open the report to see which packages dominate.

Enforce with size-limit

Use size-limit to enforce budgets in CI: 

npm install --save-dev size-limit @size-limit/preset-app

// .size-limit.json
[
  {
    "path": "dist/index.js",
    "limit": "250kb"
  },
  {
    "path": "dist/index.css",
    "limit": "50kb"
  }
]

// package.json
"scripts": {
  "size": "size-limit"
}

In CI (GitHub Actions): 

- name: Check Bundle Size
  run: npm run size

Code Splitting Strategy

Split by route to load only what's needed: 

import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
      </Routes>
    </Suspense>
  );
}

Reduce Dependencies

  • Audit: npm audit and npm ls to find unnecessary deps
  • Replace lodash with native: Modern JS eliminates most lodash use cases
  • Use lightweight alternatives: date-fns instead of moment, clsx instead of classnames
  • Remove unused packages: depcheck, unused-files-cli

Monitoring & Alerts

Track bundle size trends over time: 

// GitHub Actions workflow
- name: Report Bundle Size
  uses: andresz1/size-limit-action@v1
  with:
    github_token: ${{ secrets.GITHUB_TOKEN }}

This posts bundle size changes on pull requests so developers see the impact.

Recommended Budgets (by app type)

  • Dashboard/Internal: 400–600KB
  • SPA (public): 200–300KB
  • Marketing site: <100KB
  • Mobile web: <150KB (main chunk)

Conclusion

Performance budgets aren't about perfectionism; they're about awareness. When developers see bundle size impact on every PR, decisions improve naturally.
